## 1. 简介

Claude Agent 是一个**模块化、多 LLM 提供商、可扩展**的 Python AI Agent 框架。它遵循 **ReAct（Reasoning + Acting，推理 + 行动）** 范式，核心工作流程为：

```
用户输入 → 理解意图 → 规划步骤 → 调用工具 → 观察结果 → 反思调整 → 输出最终答案
```

### 核心能力

| 能力 | 说明 |
|------|------|
| 🔄 **自主推理** | Agent 能在"思考→行动→观察"的循环中自主解决问题 |
| 🔧 **工具调用** | 自动判断何时需要调用工具，支持并行/串行调用 |
| 🧠 **上下文记忆** | 保持多轮对话上下文，支持滑动窗口自动管理 |
| 🔌 **多提供商** | 同一套代码可切换 OpenAI（GPT-4o）或 Anthropic（Claude） |
| 📦 **工具扩展** | 自定义工具只需继承基类，放入目录即自动注册 |

## 2. 安装与环境配置

### 2.1 环境要求

- Python >= 3.10
- pip 包管理器

### 2.2 安装步骤

```bash
# 进入项目目录
cd E:\Agent\Claude_Agent

# 安装项目及依赖
pip install -e .

# 如需开发依赖（测试框架等）
pip install -e ".[dev]"
```

### 2.3 配置 API Key

```bash
# 复制环境变量模板
cp .env.example .env
```

编辑 `.env` 文件，填入你的 API 密钥：

```env
# ========== LLM 提供商选择 ==========
# 可选值：openai | anthropic
LLM_PROVIDER=openai

# ========== OpenAI 配置 ==========
OPENAI_API_KEY=sk-你的密钥
OPENAI_MODEL=gpt-4o
OPENAI_BASE_URL=https://api.openai.com/v1

# ========== Anthropic 配置 ==========
# 如果使用 Anthropic，将 LLM_PROVIDER 改为 anthropic
ANTHROPIC_API_KEY=sk-ant-你的密钥
ANTHROPIC_MODEL=claude-sonnet-4-6-20250514

# ========== Agent 行为控制 ==========
AGENT_MAX_ITERATIONS=10      # 最大推理步数
AGENT_TEMPERATURE=0.7        # 生成温度（0=确定，1=创造）
AGENT_MAX_TOKENS=4096        # 单次 LLM 调用的最大输出 Token

# ========== 记忆存储 ==========
MEMORY_DIR=./data/memory
VECTOR_STORE_DIR=./data/vector_store

# ========== 日志级别 ==========
LOG_LEVEL=INFO               # DEBUG | INFO | WARNING | ERROR
```

## 3. CLI 命令行使用

安装后提供 `claude-agent` 命令，也可使用 `python -m claude_agent`。

### 3.1 查看版本

```bash
python -m claude_agent version
# 输出: Claude Agent v0.1.0
```

### 3.2 查看帮助

```bash
python -m claude_agent --help
```

输出：

```
Usage: python -m claude_agent [OPTIONS] COMMAND [ARGS]...

  Claude Agent — A modular, multi-provider Python AI Agent.

  Use 'chat' for interactive mode or 'run' for a one-shot task.

Options:
  --version  Show the version and exit.
  --help     Show this message and exit.

Commands:
  chat      Start an interactive chat session with the Agent.
  run       Run a single task and exit.
  version   Print version info.
```

### 3.3 单次任务模式 `run`

```bash
# 基本用法：执行一个任务并输出结果
python -m claude_agent run "计算 128 * 45"

# 流式输出：实时看到 Agent 的思考和工具调用过程
python -m claude_agent run --stream "搜索今天北京的天气"

# 指定提供商
python -m claude_agent run --provider openai "Write a Python quicksort"

# 指定模型
python -m claude_agent run --model gpt-4o-mini "解释什么是递归"

# 组合使用
python -m claude_agent run --provider anthropic --model claude-sonnet-4-6-20250514 \
  "分析 app.log 中的错误"
```

### 3.4 交互式对话模式 `chat`

```bash
# 启动交互式对话
python -m claude_agent chat

# 指定提供商和模型
python -m claude_agent chat --provider openai --model gpt-4o
```

## 5. 作为 Python 库使用

### 5.1 最简示例

```python
import asyncio
from claude_agent import Agent, Settings, create_llm_provider
from claude_agent.memory.conversation import ConversationMemory
from claude_agent.tools.registry import ToolRegistry

async def main():
    # 1. 加载配置
    settings = Settings()

    # 2. 创建 LLM
    llm = create_llm_provider(settings)

    # 3. 注册工具
    tools = ToolRegistry()
    tools.discover_builtin()

    # 4. 初始化记忆
    memory = ConversationMemory(
        max_tokens=settings.conversation_max_tokens,
        system_prompt=settings.system_prompt,
    )

    # 5. 创建 Agent
    agent = Agent(llm=llm, tools=tools, memory=memory, settings=settings)

    # 6. 执行任务
    answer = await agent.run("用快速排序对 [3,1,4,1,5,9,2,6] 排序")
    print(answer)

asyncio.run(main())
```

### 5.2 流式输出

```python
async def stream_example():
    settings = Settings()
    llm = create_llm_provider(settings)
    tools = ToolRegistry()
    tools.discover_builtin()
    memory = ConversationMemory(
        system_prompt=settings.system_prompt,
    )
    agent = Agent(llm=llm, tools=tools, memory=memory, settings=settings)

    # 流式获取输出
    async for chunk in agent.run_stream("搜索 Python 3.13 新特性"):
        print(chunk, end="", flush=True)

asyncio.run(stream_example())
```

### 5.3 多轮对话

```python
async def multi_turn():
    settings = Settings()
    llm = create_llm_provider(settings)
    tools = ToolRegistry()
    tools.discover_builtin()
    memory = ConversationMemory(
        system_prompt="你是一个 Python 编程助手。",
    )
    agent = Agent(llm=llm, tools=tools, memory=memory, settings=settings)

    # 第一轮
    response1 = await agent.run("写一个二分查找函数")
    print(response1)

    # 第二轮：Agent 会记住上一轮的上下文（无需传历史）
    response2 = await agent.run("给这个函数加上类型注解")
    print(response2)

    # 第三轮
    response3 = await agent.run("写几个单元测试")
    print(response3)

    # 重置对话（开启新话题）
    agent.reset()
    response4 = await agent.run("现在帮我写一个装饰器")
    print(response4)

asyncio.run(multi_turn())
```

### 5.4 动态工具开关

```python
async def selective_tools():
    settings = Settings()
    llm = create_llm_provider(settings)
    tools = ToolRegistry()

    # 只注册安全的工具
    from claude_agent.tools.builtin.file_tools import ReadFileTool
    from claude_agent.tools.builtin.web_tools import HttpGetTool

    tools.register(ReadFileTool())
    tools.register(HttpGetTool())
    # 不注册 shell 和 python 工具 = 更安全

    memory = ConversationMemory(system_prompt=settings.system_prompt)
    agent = Agent(llm=llm, tools=tools, memory=memory, settings=settings)

    answer = await agent.run("读取 README.md 的内容并总结")
    print(answer)
```

## 6. 核心架构详解

### 6.1 ReAct 循环

Agent 的核心是一个循环，每一步包含四个阶段：

```
┌──────────────────────────────────────────┐
│                   用户输入                 │
└──────────────────┬───────────────────────┘
                   ▼
     ┌─────────────────────────┐
     │  1. 思考 (THINKING)      │  LLM 分析当前状态，决定下一步
     │     "我需要查文件..."     │
     └───────────┬─────────────┘
                 ▼
     ┌─────────────────────────┐
     │  2. 行动 (ACTING)        │  执行工具调用
     │     调用 read_file 工具   │
     └───────────┬─────────────┘
                 ▼
     ┌─────────────────────────┐
     │  3. 观察 (OBSERVING)     │  接收工具返回结果
     │     文件内容是...        │
     └───────────┬─────────────┘
                 ▼
     ┌─────────────────────────┐
     │  4. 输出 / 继续循环       │  有答案则输出，否则回到步骤 1
     └─────────────────────────┘
```

### 6.2 状态机

Agent 的状态流转：

```
IDLE → THINKING → ACTING → OBSERVING → THINKING → ... → DONE
                                  ↘ ERROR
```

通过 `agent.state` 属性可查询当前状态。

### 6.3 模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| **Core** | `core/agent.py` | ReAct 主循环，控制"思考→行动→观察"流程 |
| | `core/planner.py` | 将复杂任务分解为子步骤 |
| | `core/executor.py` | 查找并执行工具，处理异常 |
| | `core/types.py` | 核心数据模型（Message, ToolCall, ToolResult 等） |
| **LLM** | `llm/base.py` | 抽象基类 + OpenAI/Anthropic 格式互转 |
| | `llm/openai_provider.py` | OpenAI API 实现 |
| | `llm/anthropic_provider.py` | Anthropic Claude API 实现 |
| | `llm/factory.py` | 根据配置自动创建对应 Provider |
| **Tools** | `tools/base.py` | BaseTool 抽象类 + ToolError |
| | `tools/registry.py` | 工具注册表 + 自动发现 |
| | `tools/builtin/*.py` | 7 个内置工具 |
| **Memory** | `memory/conversation.py` | 短期对话记忆（滑动窗口） |
| | `memory/vector_store.py` | 长期向量记忆（ChromaDB） |
| **Config** | `config/settings.py` | Pydantic Settings 配置管理 |

### 6.4 LLM 提供商抽象

所有 LLM 提供商实现同一个接口 `BaseLLMProvider`：

```python
class BaseLLMProvider(ABC):
    async def chat(
        self,
        messages: list[Message],           # 对话历史
        tools: list[dict[str, Any]] | None, # 工具定义（OpenAI 格式）
    ) -> Message:                           # 统一返回 AssitantMessage
        ...
```

**关键设计：** Agent 核心代码不关心底层用的是 OpenAI 还是 Anthropic。两种 Provider 的内部差异（消息格式、工具定义格式、响应解析方式）全部封装在各自的实现类中。

## 8. 自定义工具开发

### 8.1 三步创建自定义工具

**第 1 步：** 在 `src/claude_agent/tools/custom/` 下创建 `.py` 文件

```python
# src/claude_agent/tools/custom/weather.py

from claude_agent.tools.base import BaseTool, ToolError

class WeatherTool(BaseTool):
    # 工具名称（LLM 用来识别和调用）
    name = "get_weather"

    # 工具描述（LLM 用来判断何时使用此工具）
    description = "获取指定城市的当前天气，返回温度、天气状况和湿度。"

    # 参数定义（JSON Schema 格式）
    parameters = {
        "type": "object",
        "properties": {
            "city": {
                "type": "string",
                "description": "城市名称，如 'Beijing'、'Shanghai'。",
            },
            "unit": {
                "type": "string",
                "enum": ["celsius", "fahrenheit"],
                "description": "温度单位，默认 celsius。",
                "default": "celsius",
            },
        },
        "required": ["city"],
    }

    async def execute(self, city: str, unit: str = "celsius") -> str:
        """执行天气查询。"""
        # 在这里调用真实的天气 API
        import httpx

        try:
            async with httpx.AsyncClient() as client:
                response = await client.get(
                    f"https://api.weather.com/v1/current",
                    params={"city": city, "unit": unit},
                )
                response.raise_for_status()
                data = response.json()
                return (
                    f"城市: {data['city']}\n"
                    f"温度: {data['temp']}°\n"
                    f"天气: {data['condition']}\n"
                    f"湿度: {data['humidity']}%"
                )
        except Exception as e:
            raise ToolError(f"天气查询失败: {e}")
```

**第 2 步：** 重启 Agent（工具会自动发现）

```bash
python -m claude_agent chat
# Agent 启动时自动扫描 tools/custom/ 目录
```

**第 3 步：** 在对话中使用

```
You › 北京今天天气怎么样？

# Agent 会自动识别用户的天气意图，调用 get_weather 工具
```

### 8.2 工具编写规范

**必须遵守：**

1. **继承 `BaseTool`**
2. **定义 `name`**（必需，全小写+下划线，如 `my_tool`）
3. **定义 `description`**（必需，清晰描述功能，这直接影响 LLM 是否会调用它）
4. **定义 `parameters`**（必需，标准的 JSON Schema `{"type": "object", "properties": {...}}`）
5. **实现 `async def execute(self, **kwargs) -> str`**（必需，返回字符串）
6. **异常使用 `ToolError`**（推荐，Agent 会把错误信息传给 LLM 使其调整策略）

**描述编写技巧：**

```python
# ❌ 差的描述：LLM 不知道何时该用
description = "Do stuff."

# ❌ 不够明确
description = "Search for things."

# ✅ 好的描述：明确告知功能、适用场景
description = (
    "Search the user's local codebase for files matching a glob pattern. "
    "Use this when the user asks to find files, locate code, or search "
    "for specific file types (e.g., 'find all Python files')."
)
```

### 8.3 通过代码注册工具

也可以在代码中手动注册（不放在 custom 目录）：

```python
from claude_agent.tools.registry import ToolRegistry
from my_tools import MyCustomTool

tools = ToolRegistry()
tools.discover_builtin()
tools.register(MyCustomTool())  # 手动注册
```

## 9. 配置参数说明

### 9.1 完整配置列表

| 环境变量 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `LLM_PROVIDER` | string | `openai` | LLM 提供商：`openai` 或 `anthropic` |
| `OPENAI_API_KEY` | string | — | OpenAI API 密钥 |
| `OPENAI_MODEL` | string | `gpt-4o` | OpenAI 模型名 |
| `OPENAI_BASE_URL` | string | — | 自定义 API 地址（如代理） |
| `ANTHROPIC_API_KEY` | string | — | Anthropic API 密钥 |
| `ANTHROPIC_MODEL` | string | `claude-sonnet-4-6-20250514` | Anthropic 模型名 |
| `AGENT_MAX_ITERATIONS` | int | `10` | 最大推理步数（防止死循环） |
| `AGENT_TEMPERATURE` | float | `0.7` | 生成温度：0=确定性，1=高创造性 |
| `AGENT_MAX_TOKENS` | int | `4096` | 单次 LLM 输出最大 Token |
| `CONVERSATION_MAX_TOKENS` | int | `128000` | 对话历史最大 Token（超出自动截断） |
| `MEMORY_DIR` | string | `./data/memory` | 记忆文件存储目录 |
| `VECTOR_STORE_DIR` | string | `./data/vector_store` | 向量存储目录 |
| `LOG_LEVEL` | string | `INFO` | 日志级别：DEBUG/INFO/WARNING/ERROR |

### 9.2 温度参数选择建议

| 温度值 | 适用场景 |
|--------|----------|
| `0.0 - 0.3` | 代码生成、数据分析、精确任务 |
| `0.4 - 0.7` | 通用对话、文档撰写、日常任务 |
| `0.8 - 1.0` | 创意写作、头脑风暴、开放式探索 |

### 9.3 模型选择建议

| 场景 | 推荐模型 |
|------|----------|
| 复杂推理、多步骤任务 | `gpt-4o` / `claude-sonnet-4-6` |
| 简单任务、降低成本 | `gpt-4o-mini` / 较小模型 |
| 超长上下文 | `gpt-4o`（128K context）/ `claude-sonnet-4-6` |
| 自主代理、大量工具调用 | `claude-sonnet-4-6`（工具调用更强） |

## 10. 记忆系统

### 10.1 短期记忆（ConversationMemory）

每次对话自动记录所有消息，超出 Token 上限时自动删除旧消息（但保留系统提示词）。

```python
from claude_agent.memory.conversation import ConversationMemory

memory = ConversationMemory(
    max_tokens=128_000,                    # 最大 Token 数
    system_prompt="你是一个友好的助手。",    # 系统提示词（始终保留）
)

# 对话中自动管理
memory.add(message)     # 添加消息，超出限制自动裁剪
memory.get_messages()   # 获取所有消息
memory.clear()          # 清空（保留系统提示词）
memory.token_count      # 查看当前 Token 数
```

### 10.2 长期记忆（VectorMemory）

基于 ChromaDB 的向量存储，用于跨会话持久化重要信息：

```python
from claude_agent.memory.vector_store import VectorMemory

vmem = VectorMemory(persist_dir="./data/vector_store")

# 存储一条记忆
doc_id = vmem.add(
    text="用户偏好用中文回复，喜欢代码带详细注释。",
    metadata={"type": "user_preference", "category": "language"},
)

# 搜索相关记忆
results = vmem.query("用户喜欢什么语言回复？", top_k=3)
for r in results:
    print(f"[{r['distance']:.2f}] {r['document']}")

# 管理
vmem.count()           # 记忆数量
vmem.delete(doc_id)    # 删除一条
vmem.clear()           # 清空全部
```

> **注意：** VectorMemory 目前在 Agent 主循环中未自动调用。你需要在自定义代码中手动将重要结论存入向量记忆，并在适当时机将相关记忆注入到对话上下文中。

## 13. 故障排除

### 13.1 安装问题

| 问题 | 解决方案 |
|------|----------|
| `pip install` 失败 | 检查 Python ≥ 3.10：`python --version` |
| `pip` 命令不存在 | 使用 `python -m pip` 代替 |
| ChromaDB 安装失败 | 单独安装：`pip install chromadb` |

### 13.2 运行时问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| Agent 不调用工具 | 工具描述不够清晰 | 优化 `description` 字段，明确告知 LLM 何时调用 |
| 回复质量差 | 温度过高 | 降低 `AGENT_TEMPERATURE` 到 0.3 |
| 死循环 | 工具总是失败 | 降低 `AGENT_MAX_ITERATIONS` 到 5 |
| Token 超限 | 对话历史太长 | 使用 `/reset` 清除历史，或降低温度 |
| API 连接超时 | 网络问题或代理 | 设置 `OPENAI_BASE_URL` 配置代理 |
| 乱码问题 | 编码不匹配 | 确保文件使用 UTF-8 编码 |

### 13.3 日志调试

```bash
# 开启 DEBUG 日志，查看 Agent 每一步的详细行为
LOG_LEVEL=DEBUG python -m claude_agent run "测试任务"
```

DEBUG 日志会显示：
- 每轮迭代的编号
- LLM 的请求和响应
- 工具调用的参数和结果
- Token 使用量变化
